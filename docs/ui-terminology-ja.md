# UI用語集（日本語統一）

Cataloga の Web UI では、以下の用語を標準表記として固定します。

## 基本用語

- Resource: `リソース`
- Dependency: `依存関係`
- Tag: `タグ`
- Type pack: `タイプパック`
- Draft change: `ドラフト変更`
- Review changes: `変更を確認`
- Save changes: `変更を保存`
- Discard: `破棄`

## 画面名

- Dashboard: `ダッシュボード`
- Resources: `リソース`
- Dependencies: `依存関係`
- Changes: `変更`
- Type packs: `タイプパック`
- Settings: `設定`
- Validation: `検証`

## 操作ラベル

- Create resource: `リソースを作成`
- Edit resource: `リソースを編集`
- Create dependency: `依存関係を作成`
- Basic tags: `基本タグ`
- Additional tags: `追加タグ`
- Environment: `環境`
- Owner: `オーナー`
- Site: `サイト`
- Zone: `ゾーン`
- Lifecycle: `ライフサイクル`
- Note: `補足`
- Todo: `TODO`
- Risk: `注意`
- Runs on: `実行先`
- Uses: `利用先`
- Exposure path: `公開経路`
- Belongs to: `所属先`
- Connects to: `接続先`
- Install type pack: `タイプパックをインストール`
- Enable type pack: `タイプパックを有効化`
- Disable type pack: `タイプパックを無効化`

## 状態ラベル

- Draft: `ドラフト`
- Ready: `保存可能`
- Saved: `保存済み`
- Discarded: `破棄済み`
- Valid: `正常`
- Warning: `警告`
- Error/Invalid: `エラー` / `異常`

## UIで避ける用語

通常UIでは以下を使わない:

- `Entity`
- `Relation`
- `Mutation`
- `Change Session`
- `DomainPack`

これらは内部実装・技術文脈でのみ許可します。
