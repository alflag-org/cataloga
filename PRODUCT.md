# Product

## Register

product

## Users

Cataloga is for operators and engineers who maintain an infrastructure catalog. They use it while inspecting Resource relationships, validating changes, importing or exporting catalog data, and taking Snapshots of the current state.

## Product Purpose

Cataloga is a Rust-first, schema-driven infrastructure catalog. It keeps canonical runtime data in SQLite or D1-backed storage, with YAML reserved for Import, Export, and Snapshot workflows. Success means users can understand Resource Types, Fields, Views, Relations, Draft changes, and validation results without losing trust in the catalog.

## Brand Personality

Calm, precise, operational.

## Anti-references

Avoid marketing-style dashboards, decorative graph art, dense hairball network views, and editor-like node canvases when the user only needs to inspect Relations. Do not use legacy Entity terminology in user-facing contexts.

## Design Principles

- Search first, then explore nearby Relations.
- Treat graphs as projection Views of catalog data, not as canonical storage.
- Keep dashboard surfaces compact and task-focused.
- Preserve standard product UI affordances over novelty.
- Make Validate, Save, Discard, Import, Export, and Snapshot flows feel trustworthy.

## Accessibility & Inclusion

Target keyboard-accessible product workflows, visible focus states, readable contrast, and reduced-motion-safe interactions. Graph exploration should remain usable through search, filters, and Resource detail navigation, not only pointer gestures.
